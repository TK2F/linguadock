# Contributing to LinguaDock

Thank you for your interest in contributing to LinguaDock!

## How to Contribute

### Reporting Issues

- Use the GitHub Issues page
- Describe the problem clearly
- Include browser version and OS

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

### Code Style

- Follow existing code patterns
- Use TypeScript
- Run `npm run lint` before committing

### Adding Vocabulary

- Submit CSV files following the format in README
- Ensure Polish text uses correct special characters (ą ę ó ś ć ż ź ł ń)
- Include accurate katakana pronunciation

## Development Setup

```bash
npm install
npm run dev
```

# コントリビューションガイド

LinguaDock へのご貢献に感謝します！

## 貢献方法

### 問題の報告

- GitHub Issues を使用してください
- 問題を明確に記述してください
- ブラウザバージョンとOSを含めてください

### プルリクエスト

1. リポジトリをフォーク
2. フィーチャーブランチを作成 (`git checkout -b feature/amazing-feature`)
3. 変更をコミット (`git commit -m 'Add amazing feature'`)
4. ブランチにプッシュ (`git push origin feature/amazing-feature`)
5. プルリクエストを開く

### コードスタイル

- 既存のコードパターンに従う
- TypeScript を使用
- コミット前に `npm run lint` を実行

### 単語の追加

- README の形式に従った CSV ファイルを提出
- ポーランド語は正しい特殊文字を使用 (ą ę ó ś ć ż ź ł ń)
- 正確なカタカナ発音を含める
